1. Introduction

Le projet "Heart Green Event" consiste à créer un site web professionnel pour une entreprise spécialisée dans l'organisation d'événements personnalisés et écologiques, tels que les mariages, anniversaires et baptêmes. L'objectif principal est de développer une plateforme en ligne qui facilite l'interaction entre l'entreprise et ses clients, en offrant des fonctionnalités telles que la réservation en ligne, la communication directe et la personnalisation des thèmes d'événements.

Ce projet s'inscrit dans le cadre de ma certification en tant que développeur web et vise à démontrer les compétences acquises durant ma formation. Le site web a été développé en utilisant les technologies HTML, CSS, JavaScript et PHP, avec une base de données MySQL. Il reflète l'engagement de l'entreprise envers la durabilité et la créativité, tout en offrant une expérience utilisateur fluide et sécurisée.

3. Analyse du Besoin

3.1 Présentation de l'Entreprise

Heart Green Event est une entreprise spécialisée dans l'organisation d'événements personnalisés à thème, tels que les mariages, anniversaires, baptêmes et autres célébrations. L'entreprise vise à offrir des expériences uniques en s'appuyant sur des valeurs écoresponsables. En utilisant des éléments de décoration fabriqués à partir de matériaux recyclés, Heart Green Event se positionne sur le marché en tant qu'entreprise innovante, offrant des services de qualité tout en respectant l'environnement.

Ce projet est réalisé dans le cadre d'une certification de développeur web et a pour objectif de présenter un site web fonctionnel permettant de gérer efficacement la prise de contact, la réservation d'événements et la présentation des services.

3.2 Contexte et Besoins

Le besoin principal d'Heart Green Event est de créer une présence en ligne qui permette de mieux servir ses clients et de promouvoir ses valeurs écoresponsables. En tant qu'entreprise à la recherche de solutions innovantes pour l'organisation d'événements, le site doit répondre aux besoins suivants :

  • Faciliter la prise de contact : Le site doit permettre aux clients potentiels de contacter facilement l'entreprise pour obtenir des informations sur les services, discuter des détails de l'événement ou demander des devis.
  • Réservation en ligne : Permettre aux clients de réserver une date et un type d'événement directement sur le site web. Cela inclut un calendrier de disponibilité et la gestion des réservations.
  • Mise en avant des thèmes : Offrir une galerie des thèmes disponibles pour que les clients puissent choisir un thème d'événement préconçu.

    Insertion recommandée : Inclure des images des différents thèmes proposés, ce qui permettra de donner un aperçu visuel aux clients potentiels.

    Image à insérer : Galerie des thèmes (fichiers "image.png" téléchargés).

  • Renforcer l'image de marque : Le site doit refléter les valeurs écoresponsables de l'entreprise en utilisant une charte graphique en accord avec la thématique écologique, ainsi que des illustrations et des couleurs évoquant la nature.

Le site web vise à offrir une expérience utilisateur fluide et intuitive, en rendant la navigation simple, les informations faciles à trouver et le processus de réservation sans friction. De plus, il est important d'améliorer la visibilité en ligne de l'entreprise grâce à l'optimisation SEO (Search Engine Optimization).

4. Arborescence du Site

L'arborescence du site est conçue pour permettre une navigation claire et intuitive, facilitant l'accès à l'information pour les utilisateurs. La structure générale est la suivante :

  • Accueil : Page principale présentant les services de Heart Green Event et les principales sections du site. Cette page contient également des informations sur les valeurs de l'entreprise, la mise en avant des événements proposés et des boutons de redirection rapide vers les sections importantes.
  • Nos Réalisations : Cette section permet de présenter des exemples d'événements organisés par Heart Green Event. Chaque réalisation est illustrée par des photos et des descriptions mettant en avant le thème, la décoration et l'impact écologique de l'événement.
  • Célébrer : Section dédiée aux offres de célébrations personnalisées proposées par l'entreprise. Elle inclut :
    • Les offres : Présentation des différents types de célébrations proposées (mariages, anniversaires, baptêmes, etc.).
    • Questionnaire : Un questionnaire destiné aux clients pour mieux comprendre leurs attentes et les aider à définir leur événement personnalisé.
  • Planning : Page permettant de visualiser les créneaux disponibles pour réserver un événement. Les clients peuvent vérifier la disponibilité et réserver une date en fonction des créneaux restants.
  • Formulaire/Contact : Page de contact permettant aux utilisateurs de poser des questions, de demander des informations supplémentaires ou de solliciter un devis personnalisé pour leurs événements. Un formulaire est mis à disposition pour faciliter la prise de contact.
  • Connexion : Section permettant aux utilisateurs de se connecter à leur compte personnel. Une fois connectés, les utilisateurs ont accès aux fonctionnalités suivantes :
    • Utilisateur : Gestion des informations personnelles, modification des coordonnées et visualisation des événements réservés.
    • Votre Événement : Espace où l'utilisateur peut suivre l'évolution de l'organisation de son événement, accéder aux informations partagées par l'équipe de Heart Green Event et communiquer avec eux.
    • Discussions : Fonctionnalité de messagerie permettant de communiquer directement avec l'équipe organisatrice et de poser des questions spécifiques liées aux événements en cours.

7. Conception

7.1 Modèle de Données Conceptuel (MDC)

Le Modèle de Données Conceptuel (MDC) est la première étape de la conception de la base de données. Il identifie les entités clés du système ainsi que les relations entre elles. Pour "Heart Green Event", les principales entités identifiées sont les suivantes :

  • Utilisateur : Cette entité représente les clients et administrateurs du site, avec des attributs tels que le nom, prénom, email et mot de passe.
  • Adresse : Un utilisateur peut posséder une ou plusieurs adresses. L'adresse est détaillée par pays, ville et code postal.
  • Message : Permet la communication entre les utilisateurs (clients, administrateurs). Chaque message comporte un contenu, un expéditeur et un destinataire.
  • Commentaire : Les utilisateurs peuvent ajouter des commentaires sur les réalisations, avec des informations telles que la date de création.
  • Réalisation : Représente les événements déjà organisés par Heart Green Event, servant d'exemple pour les clients potentiels. Les réalisations peuvent contenir des photos et des vidéos.
  • Historique de connexion : Stocke les informations relatives aux connexions des utilisateurs, telles que l’adresse IP et la date de connexion.
  • Rôle : Chaque utilisateur est associé à un rôle (ex. client, administrateur).

Les relations entre ces entités permettent de définir les interactions possibles. Par exemple, un utilisateur peut résider dans une ville spécifique, envoyer plusieurs messages ou être lié à une réalisation par des commentaires.

7.2 Modèle Logique de Données (MLD)

Le Modèle Logique de Données (MLD) est la version normalisée du MDC. Il est destiné à être traduit en schéma de base de données physique (c'est-à-dire les tables réelles). Il présente les tables principales du système et décrit leurs relations. Voici les principales tables pour "Heart Green Event" :

  • Utilisateur (users) :
    • Contient des attributs tels que Nom, Prénom, Email et mdp (mot de passe, qui est chiffré avant d'être stocké).
  • Adresse (address) :
    • Stocke les adresses des utilisateurs, avec des colonnes pour le pays (Id_Pays), la ville (Id_Ville) et le code postal (Id_Code_postal).
  • Réalisation (realisation) :
    • Table contenant les réalisations d'événements organisés par Heart Green Event, avec des descriptions et des liens vers des photos/vidéos.
  • Message (message) :
    • Cette table permet de stocker les messages échangés entre utilisateurs et administrateurs (Id_Expéditeur, Id_Destinataire, Contenu, date_envoi).
  • Commentaire (commentaire) :
    • Stocke les commentaires des utilisateurs sur les différentes réalisations, avec des attributs comme Contenu et date_création.
  • Historique de connexion (historique_de_connexion) :
    • Stocke des informations sur chaque connexion utilisateur, incluant Date_de_connexion et Adresse_IP.
  • Rôle (role) :
    • Cette table contient les rôles associés aux utilisateurs (par exemple, client ou administrateur).

Relations entre les tables :

  • Utilisateur - Adresse : Un utilisateur peut avoir plusieurs adresses, mais chaque adresse est associée à un utilisateur spécifique.
  • Utilisateur - Message : Un utilisateur peut envoyer et recevoir des messages. Ces messages sont reliés par des clés étrangères (Id_Expéditeur, Id_Destinataire).
  • Utilisateur - Réalisation : Un utilisateur peut être l'auteur d'une réalisation ou un spectateur qui peut commenter.
  • Utilisateur - Historique de connexion : Un utilisateur peut avoir un historique complet de ses connexions, incluant des détails comme l'IP et la date.

Maquettage des Interfaces

Le processus de maquettage a débuté par la conversion des wireframes en pages web statiques. Cette étape a permis de définir clairement la structure visuelle et l'agencement des différents éléments du site. En adoptant une approche mobile-first, j'ai assuré que le site soit parfaitement adapté aux écrans de petite taille avant de le rendre compatible avec des dispositifs plus grands.

Pour ce faire, j'ai utilisé les technologies Flexbox et Grid de CSS, qui offrent une grande flexibilité pour la création de mises en page réactives. Par exemple, le fichier CSS commence par un reset CSS d'Eric Meyer, garantissant une base cohérente pour le stylisme de tous les éléments HTML :

/* CSS Version Mobile First */

}

	flex-direction: column;

	align-items: center;
}

/* Autres styles suivent... */

Cette méthode a permis de normaliser les styles par défaut des navigateurs et de construire une base solide pour le design personnalisé du site.

Développement des Interfaces Dynamiques

Après avoir établi les pages statiques, j'ai concentré mes efforts sur l'ajout d'interactivité et d'animations pour enrichir l'expérience utilisateur. L'un des défis majeurs a été la création d'un effet de scroll fluide et sans bugs, garantissant que les éléments s'animent de manière harmonieuse lorsqu'ils entrent dans le champ de vision de l'utilisateur.

Fonctionnalité de Scroll Animations

L'implémentation de cette fonctionnalité a nécessité une approche méthodique pour s'assurer que les animations se déclenchent correctement sans interférer avec la performance globale du site. Voici comment j'ai procédé :

  1. Sélection des Éléments à Animer : J'ai commencé par identifier tous les éléments susceptibles de recevoir une animation au défilement. Cela inclut les divs, images, paragraphes, et boutons présents dans le contenu principal.
    let scrollElements = document.querySelectorAll(
	"main div, main img, main p, main .nousContacter, main h1, label, input, textarea, button, main a, #connexion"
);
  1. Détection de la Visibilité : Pour déterminer si un élément est visible dans la fenêtre du navigateur, j'ai créé deux fonctions : elementInView et elementOutofView. Ces fonctions vérifient la position de l'élément par rapport à la hauteur de la fenêtre.
    function elementInView(element) {
	let elementTop = element.getBoundingClientRect().top;
	let viewportHeight = window.innerHeight;
	return elementTop < viewportHeight;
}

function elementOutofView(element) {
	let elementTop = element.getBoundingClientRect().top;
	let viewportHeight = window.innerHeight;
	return elementTop > viewportHeight;
}
  1. Application des Animations : Ensuite, une fonction handleScrollAnimation parcourt tous les éléments sélectionnés et applique la classe scroll lorsqu'ils entrent dans la vue. Cette classe déclenche les animations définies dans le CSS.
    function handleScrollAnimation() {
	scrollElements.forEach(element => {
		if (elementInView(element)) {
			element.classList.add("scroll");
			element.classList.add("already-shown");
		} else if (elementOutofView(element) && !element.classList.contains("already-shown")) {
			element.classList.remove("scroll");
		}
	});
}

window.addEventListener("scroll", handleScrollAnimation);
window.addEventListener("load", handleScrollAnimation);
  1. Définition des Animations en CSS : La classe scroll est associée à des animations CSS qui définissent comment les éléments doivent apparaître lorsqu'ils sont scrolled dans la vue.
    /* Animation de fadeIn pour les éléments en scroll */
.scroll {
    animation: fadeIn 1s ease-in-out both;
}

@keyframes fadeIn {
    0% {
        transform: translateY(100px);
        opacity: 0;
    }
    100% {
        transform: translateY(0);
        opacity: 1;
    }
}

Cette fonctionnalité a apporté une dimension supplémentaire au site, rendant la navigation plus engageante et visuellement plaisante pour les utilisateurs.

10. Développement Back-End

Le développement back-end de Heart Green Event a été pensé pour fournir une gestion efficace des utilisateurs, une communication interne fluide via la messagerie et une architecture sécurisée. L'objectif a été de construire une base solide pour le site en se concentrant sur l'authentification des utilisateurs, la récupération des données et la messagerie interne. La structure repose sur PHP avec PDO pour les interactions avec une base de données MySQL.

Mise en Place de la Base de Données

La base de données du projet est au cœur des opérations back-end. La gestion des utilisateurs, des messages et des autres entités est centralisée et organisée de manière à assurer une haute performance et une sécurité optimale.

  • Table Utilisateur : Cette table gère les informations personnelles des utilisateurs, comme leurs noms, prénoms, adresses e-mail, et mots de passe chiffrés.
  • Table Message : Elle contient les messages échangés entre les utilisateurs, avec les informations de l'expéditeur et du destinataire.

  1. Messagerie Interne

Le modèle MessageModel gère la fonctionnalité de messagerie. Il permet aux utilisateurs d'envoyer des messages, de les récupérer et de les afficher dans l'ordre d'envoi. La messagerie est essentielle pour la communication interne entre les clients et l'équipe de Heart Green Event.

  • Récupération des Messages : Voici un exemple de code pour récupérer tous les messages échangés entre deux utilisateurs :
public function getMessagesBetweenUsers($userId, $otherUserId) {
    $query = $this->db->prepare("
        SELECT * FROM Message
        WHERE (Id_Expéditeur = :userId AND Id_Destinataire = :otherUserId)
        OR (Id_Expéditeur = :otherUserId AND Id_Destinataire = :userId)
        ORDER BY date_envoi ASC
    ");
    $query->execute([
        'userId' => $userId,
        'otherUserId' => $otherUserId
    ]);
    return $query->fetchAll(PDO::FETCH_ASSOC);
}
  • Envoi de Messages : Voici la méthode utilisée pour insérer un nouveau message dans la base de données :
public function insertMessage($content, $expediteurId, $destinataireId) {
    if (empty($content) || !is_numeric($expediteurId) || !is_numeric($destinataireId)) {
        throw new InvalidArgumentException("Données invalides pour l'envoi du message.");
    }

    $query = $this->db->prepare("
        INSERT INTO Message (Contenu, date_envoi, Id_Expéditeur, Id_Destinataire)
        VALUES (:content, NOW(), :expediteurId, :destinataireId)
    ");
    $query->execute([
        'content' => $content,
        'expediteurId' => $expediteurId,
        'destinataireId' => $destinataireId
    ]);
}

Ce module de messagerie assure une communication fluide entre les différentes parties et enregistre chaque message avec son contenu, l'heure d'envoi et les utilisateurs concernés.

Sécurité

La sécurité des données et des utilisateurs a été une priorité lors de la phase de développement back-end. Plusieurs mesures ont été prises pour garantir que le site soit protégé contre les attaques courantes :

  1. Chiffrement des Mots de Passe : Tous les mots de passe des utilisateurs sont chiffrés avant d'être stockés dans la base de données. La fonction PHP password_hash avec l'algorithme bcrypt est utilisée à cette fin. Cela assure que les mots de passe ne sont jamais stockés en clair dans la base de données, garantissant ainsi une meilleure protection contre les failles potentielles.
  1. Protection contre les Injections SQL : Grâce à l'utilisation de PDO, toutes les interactions avec la base de données sont sécurisées par l'utilisation de requêtes préparées, ce qui protège contre les attaques par injection SQL.

    Par exemple, voici une requête sécurisée pour la recherche d'un utilisateur via son prénom :

    public function getUserByPrenom($prenom) {

11. Documentation du Déploiement

Guide de Déploiement Détaillé

Le déploiement du site Heart Green Event a été réalisé sur un serveur LAMP (Linux, Apache, MySQL, PHP) hébergé sur un Raspberry Pi. Cette section décrit les étapes clés que j'ai suivies pour configurer le serveur, lier le domaine heartgreenevent.fr via OVH, sécuriser le site avec Certbot, et déployer le code source organisé selon le modèle MVC avec les dossiers app, config et public.

1. Configuration du Serveur

1.1. Mise à Jour du Système

Pour garantir la stabilité et la sécurité de mon serveur, j'ai commencé par mettre à jour les paquets existants. Cela a été effectué en exécutant la commande suivante dans le terminal :

sudo apt update && sudo apt upgrade -y

Cette commande a permis de s'assurer que tous les logiciels installés étaient à jour avec les dernières versions disponibles.

1.2. Installation d'Apache

Apache a été choisi comme serveur web pour héberger le site. L'installation s'est faite rapidement grâce à la commande :

sudo apt install apache

Après l'installation, Apache démarre automatiquement. Pour vérifier son bon fonctionnement, j'ai accédé à l'adresse IP de mon Raspberry Pi via un navigateur web, ce qui a affiché la page par défaut d'Apache, confirmant ainsi que le serveur web fonctionnait correctement.

1.3. Installation de PHP

PHP est essentiel pour le traitement des scripts côté serveur. J'ai installé PHP ainsi que les modules nécessaires pour l'intégration avec Apache en utilisant la commande suivante :

sudo apt install php libapache2-mod-php

Cette étape a permis à Apache de traiter les fichiers PHP, rendant possible la génération dynamique des pages web.

1.4. Installation de MySQL

Pour la gestion des données de l'application, j'ai installé MySQL avec la commande :

sudo apt install mysql-server -y

Bien que la gestion détaillée de la base de données ne soit pas abordée ici, cette installation est cruciale pour stocker et manipuler les données nécessaires au fonctionnement du site.

1.5. Sécurisation de MySQL

La sécurisation de MySQL est une étape importante pour protéger les données. J'ai exécuté le script de sécurisation avec :

sudo mysql_secure_installation

Cela m'a permis de définir un mot de passe root sécurisé, de supprimer les utilisateurs anonymes, de désactiver les connexions root à distance et de supprimer la base de données de test, renforçant ainsi la sécurité de mon serveur de base de données.

2. Configuration des Virtual Hosts

2.1. Création du Répertoire du Site

sudo mkdir -p /var/www/heartgreenevent.fr/public_html

Ce répertoire servira de racine pour le contenu public du site.

sudo chown -R $USER:$USER /var/www/heartgreenevent.fr/public_html
sudo chmod -R 755 /var/www

Ces commandes assurent que les fichiers sont accessibles et modifiables par moi tout en maintenant des permissions sécurisées.

2.3. Création d'une Page d'Index de Test
